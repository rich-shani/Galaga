QUICK WIN #3: Added Recursion Depth Limit to Rogue Spawning
================================================================================
Status: ✅ COMPLETE
File: scripts/GameManager_STEP_FNs/GameManager_STEP_FNs.gml (lines 495-539)

CHANGES:
• Added _depth parameter with default value 0
• Added depth limit check: MAX_ROGUE_SPAWN_DEPTH = 16
• Added bounds checking for spawn array index
• Enhanced error logging with detailed messages

BEFORE:
    function spawnRogueEnemy(_spawn) {
        var enemy_data = spawn_data.PATTERN[...].WAVE[...].SPAWN[_spawn];
        // ... spawn code ...
        if (enemy_data.COMBINE) {
            spawnRogueEnemy(_spawn+1);  // Could recurse infinitely
        }
    }

AFTER:
    function spawnRogueEnemy(_spawn, _depth = 0) {
        if (_depth > 16) {
            log_error("spawnRogueEnemy exceeded max recursion depth", ...);
            return;  // Prevents stack overflow
        }
        if (_spawn < 0 || _spawn >= array_length(spawn_array)) {
            log_error("spawnRogueEnemy spawn index out of bounds", ...);
            return;  // Prevents array access crash
        }
        // ... spawn code ...
        if (enemy_data.COMBINE) {
            spawnRogueEnemy(_spawn + 1, _depth + 1);  // Track depth
        }
    }

SAFETY IMPROVEMENTS:
• Prevents stack overflow from infinite COMBINE chains
• Prevents array index out-of-bounds crashes
• Detects malformed JSON with circular spawn data
• Max depth 16 allows reasonable paired spawning

TECHNICAL DETAILS:
• Recursion type: Tail recursion (could be looped)
• Max depth: 16 (allows 8 pairs: conservative and safe)
• Error handling: Logs and returns gracefully
• Testing: Test with invalid JSON to verify error handling
