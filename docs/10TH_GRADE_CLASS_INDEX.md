## Learning Objectives

After completing this material, students will:

1. **Understand state machines**
   - [ ] Define what a state machine is
   - [ ] Draw state diagrams
   - [ ] Recognize in everyday examples

2. **Understand inheritance**
   - [ ] Explain the DRY principle
   - [ ] Recognize when inheritance is useful
   - [ ] Design inheritance hierarchies

3. **Understand data-driven design**
   - [ ] Contrast hardcoded vs. configurable approaches
   - [ ] Understand JSON structure basics
   - [ ] Recognize benefits

4. **Understand error handling**
   - [ ] Recognize defensive programming
   - [ ] Explain graceful degradation
   - [ ] Understand production code robustness

5. **Understand professional code quality**
   - [ ] Recognize design patterns
   - [ ] See importance of testing
   - [ ] Compare to beginner code

## Common Questions Answered

### "Do I need to know GameMaker to teach this?"
**No!** The lesson focuses on programming concepts, not GameMaker syntax.

### "Can I teach this without running the game?"
**Yes!** Use screenshots and the lecture script. (Running the game is nice bonus.)

### "What if students ask technical questions about the code?"
**Good!** This means they're engaged. Either:
- Answer if you know
- Say "Great question - let's look at the code later"
- Have them research and report back

### "How do I handle students who finish activities early?"
- Challenge questions (see Activities)
- Code review exercises
- Design their own concept
- Help peers

### "Can I adapt this for different ages?"
**Yes!**
- Younger (middle school): Simplify, more visuals, less code
- Older (AP/IB): More depth, more technical details, harder challenges

## Troubleshooting

### If Students Seem Lost
- Slow down
- Go back and re-explain
- Use different example
- Do an activity to clarify

### If You're Running Behind
- Skip some pattern details (Segment 5)
- Shorten code walkthrough (Segment 6)
- Do activities as homework
- Extend to next day

### If Technology Fails
- Use printed slides
- Draw on whiteboard
- Summarize from memory
- Pre-record video as backup

### If One Student Dominates
- Direct other questions to different students
- "Let's hear from someone else"
- Pair them with someone for deeper work
- Celebrate their enthusiasm
