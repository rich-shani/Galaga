## 🎓 Key Concepts Explained

### State Machines
**What:** A system in ONE state at a time with clear transitions
**Example:** Traffic light (Red → Yellow → Green)
**Why:** Prevents invalid combinations, organizes complex logic
**Time:** 5 minutes to understand

### Inheritance
**What:** Child classes inherit from parent class (code reuse)
**Example:** TieFighter, Interceptor, Shuttle inherit from Enemy
**Why:** Eliminates duplicate code (DRY principle)
**Savings:** 65% code reduction in this example

### Data-Driven Design
**What:** Configuration in JSON, not hardcoded in code
**Example:** Enemy waves defined in wave_spawn.json
**Why:** Change without recompiling (1 second vs. 30 seconds)
**Benefit:** Non-programmers can modify content

### Error Handling
**What:** Safe functions that return defaults instead of crashing
**Example:** safe_get_asset("enemy_name", default_value)
**Why:** Game continues with reduced functionality
**Benefit:** Professional quality

### Enums
**What:** Named constants instead of magic numbers
**Example:** GameMode.GAME_ACTIVE instead of 3
**Why:** Self-documenting, prevents errors
**Benefit:** Code is readable to humans
